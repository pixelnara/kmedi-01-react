현재 개발 완료된 메인 페이지를 기준으로 디자인 시스템 토큰을 현행화해주세요.

목적은 새로운 디자인 시스템을 만드는 것이 아니라,
현재 구현된 웹사이트의 실제 토큰 구조를 정리하고 문서화하는 것입니다.

# 핵심 전제

현재 웹사이트는 theme.css의 아래 영역부터 시작해서 제작되었습니다.

```css
/* ---------- SEMANTIC (alias) ---------- */
```

즉, 현재 구현된 메인 페이지의 실제 기준은 primitive token이 아니라 semantic token입니다.

중요

현재 웹사이트는 Semantic Token을 기준으로 구현되었습니다.

따라서 현재 UI 분석, 컴포넌트 분석, 토큰 사용 분석은
Semantic Token을 기준으로 진행해주세요.

Primitive Token은 디자인 시스템의 Foundation Layer로 유지합니다.

다만 Primitive Token의 값은
초기 디자인 원본이 아니라
Semantic Token을 정리하는 과정에서 구성된 Reference Layer입니다.

따라서

- 디자인 해석 기준은 Semantic Token
- 값 저장 및 참조 구조는 Primitive Token

으로 이해해주세요.

분석 우선순위

1. Semantic Token
2. Component Usage
3. Primitive Token Reference

순서로 분석해주세요.

Primitive Token을 기준으로
색상 체계를 재구성하거나
디자인 의도를 추론하지 마세요.

현재 구현된 웹사이트가
유일한 Source of Truth 입니다.

# 작업 원칙

1. Semantic Token First

현재 웹사이트 분석은 반드시 semantic token부터 시작합니다.

- --background
- --foreground
- --primary
- --secondary
- --accent
- --surface-elevated
- --text-body
- --text-headline
- --text-subtle
- --border
- --card
- --muted
- --ring
- --font-display
- --font-body
- --radius

등 실제 UI에 사용되는 semantic token을 기준으로 분석해주세요.

2. Primitive Token의 역할

primitive token은 semantic token을 설명하기 위한 참조값입니다.

- primitive를 디자인 판단 기준으로 삼지 말 것
- primitive를 임의 확장하지 말 것
- semantic token에 연결된 값만 정리
- 사용되지 않는 primitive는 별도로 표시

3. 현재 구현 기준 유지

현재 메인 페이지의 시각 방향을 유지합니다.

- 레이아웃 변경 금지
- 컴포넌트 구조 변경 금지
- 폰트 변경 금지
- 컬러를 새로 제안하지 말 것
- 디자인 톤을 새로 해석하지 말 것

# 분석 작업

## 1. OLD-Dev-Token 구조 분석

OLD-Dev-Token 폴더 안의 파일들을 먼저 분석해주세요.

확인할 것:

- 파일 구성
- 토큰 네이밍 규칙
- primitive / semantic 구분 방식
- theme.css 작성 방식
- CSS 변수 구조
- 기존 문서화 방식

이 구조를 기준으로 현재 웹사이트 토큰을 정리해주세요.

## 2. 현재 메인 페이지 분석

다음 파일에서 실제 사용 중인 토큰을 확인해주세요.

- index.html
- styles.css
- theme.css
- app.js

확인할 것:

- 실제 사용 중인 CSS 변수
- 직접 입력된 hex/rgb 색상
- font-family 사용 방식
- border-radius 값
- shadow 값
- background/surface/text/border/accent 적용 위치
- component별 token 사용 현황

## 3. Semantic Token 정리

현재 메인 페이지에서 실제 사용 중인 semantic token을 정리해주세요.

각 token마다 아래 정보를 작성해주세요.

- Token name
- Current value
- Referenced primitive
- Usage location
- Component role
- Keep / Remove / Review 여부

예시:

| Semantic Token | Value           | Primitive Ref | Usage           | Status |
| -------------- | --------------- | ------------- | --------------- | ------ |
| --background   | var(--gray-50)  | #f6f7f8       | page background | Keep   |
| --foreground   | var(--gray-900) | #1a1d23       | default text    | Keep   |

## 4. Primitive Token 정리

primitive token은 semantic token의 참조값으로만 정리해주세요.

다음 기준으로 분류해주세요.

- semantic에서 참조 중인 primitive
- semantic에서 참조하지 않는 primitive
- 중복 가능성이 있는 primitive
- AI 역추적으로 생성된 것으로 보이는 primitive
- 유지할 primitive
- 제거 검토할 primitive

## 5. Hard-coded Value 확인

styles.css 또는 index.html 안에 CSS variable이 아닌 직접 입력된 값이 있다면 목록화해주세요.

예시:

```css
color: #222;
background: rgba(...);
border: 1px solid #eee;
```

각 항목에 대해 아래처럼 정리해주세요.

- 현재 값
- 사용 위치
- 대응 가능한 semantic token
- 자동 치환 가능 여부
- 수동 검토 필요 여부

단, 임의로 자동 치환하지 말고 먼저 매핑표를 작성해주세요.
