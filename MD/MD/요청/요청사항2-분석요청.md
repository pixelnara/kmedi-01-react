## 9. Responsive Behavior

현재 Main Page의 반응형 구조를 분석합니다.

반드시 실제 코드 기준으로 확인하세요.

확인 항목

- 사용 중인 breakpoint 값
- media query 기준
- desktop / tablet / mobile 전환 지점
- container max-width
- grid가 column으로 바뀌는 지점
- navigation이 mobile menu로 바뀌는 지점
- hero layout이 변경되는 지점
- card / section / CTA 레이아웃 변경 방식
- font-size / spacing이 줄어드는 기준
- 이미지 비율 변화
- mobile에서 숨김 / 노출되는 요소

분석 결과는 아래 형식으로 정리하세요.

| 구분    |     기준 width | 적용 내용      | 관련 CSS 위치        |
| ------- | -------------: | -------------- | -------------------- |
| Desktop | 실제 코드 기준 | 실제 적용 내용 | 실제 파일 / selector |
| Tablet  | 실제 코드 기준 | 실제 적용 내용 | 실제 파일 / selector |
| Mobile  | 실제 코드 기준 | 실제 적용 내용 | 실제 파일 / selector |

주의

임의로 breakpoint를 새로 만들지 마세요.

현재 구현된 코드에서 실제 사용 중인 breakpoint만 정리하세요.
